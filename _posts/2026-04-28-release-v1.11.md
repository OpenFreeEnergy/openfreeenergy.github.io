We are pleased to announce the release of **openfe** version 1.11.0!

This release brings along some long-awaited features, notably support for membranes! 

Please see below for some highlights and see the [Changelog][] for more details.

## New Features

### Support for Protein-Membrane Systems

**openfe** can now perform simulations of proteins in membranes. These simulations require additional preparation: the user must supply a fully built, solvated, pre-equilibrated system, prepared with an external package. One the CLI, use the option `--protein-membrane` instead of `--protein` if you are providing this type of system, and see the new [tutorial] on RBFE Calculations of Protein-Membrane Sytems for guidance.

### Updated Default Settings for Improved Accuracy

The default value for the Hybrid TopologyProtocol setting turn_off_core_unique_exceptions has been changed to True. This means 1-4 interactions involving the unique alchemical atoms and core regions will now be interpolated on/off accordingly by default. We have updated this default based on internal research that shows that it improves the accuracy of results in most cases. See the [Relative Hyrbid Topology Protocol Documentation][] for more details.

## Installing `openfe v1.11.0`

`openfe` is made available as a [conda-forge][] package, or [docker and singularity images][].

Please see our [installation instructions][] for details on how to install `openfe`.
